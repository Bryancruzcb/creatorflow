Device-evidence matrix

The Stress Lab proves CreatorFlow can contain difficult fixtures in the browser. That is not the same as proving they run on a Roblox client. This matrix keeps the two apart on purpose: modeled cells are what CreatorFlow measured in-browser (solid); measured cells are Studio and physical-device results (dashed) — and most are still open, because a transfer size is a hypothesis about runtime, never a substitute for it.

Reading the matrix. Solid cells are evidence CreatorFlow holds today; dashed cells are claims that only a Studio import or a real device can settle, and a dashed cell staying pending is information, not an omission — it marks exactly where a modeled result must not be read as a runtime result. The dense FBX row is the honest case: CreatorFlow can already say it is over the polygon budget (modeled), but whether it actually tanks frame pacing on a mid-tier phone is a measured question the lab does not answer. Filling the measured columns is the manual device pass in the friend-test runbook.
